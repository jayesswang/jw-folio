INTRODUCTION

If you were asked to complete the following analogy, how would you answer?

Architect is to engineer,
as web designer is to ________________________.

If you responded with "web developer" (or anything similar), then you understand that these pairings describe the professional relationship between one who creates an idea and one who converts that idea into a tangible final product.

INSPIRATION

Recently, I discovered a series of YouTube videos by Google Chrome Developers titled, “Designers vs. Developers.” In each episode, the hosts discuss common industry pain points by opening a conversation between the two sides, providing takeaways, solutions to workflows, tools, and discussions on everyday struggles. With these insights into the importance and difficulties in the designer/developer relationship, I felt intrigued to study the development process in converting mockups into a functional website and inspired to level up my ability to empathize and communicate with others.

In this project, I challenged myself to (literally!) step into a developer's shoes by hand coding a responsive website of my portfolio from scratch and hosting it on GitHub. I’m always open to growing as a designer, so this project gave me the opportunity to try something new, test out ideas, and expand my range of skills.

TECHNICAL INSIGHTS

Starting with the basics, I studied in-depth how the CSS box model works: that it is essentially a box that wraps around every HTML element, and that it consists of margins, borders, padding, and the actual content. I enjoyed reading about the history of HTML/CSS and felt immense gratitude for the introduction of CSS Grid and Flexbox. I can't imagine the coding acrobatics developers in the past went through just to create the proper website layout!

The HTML below showcases how I applied the box model to create a centered and fixed navigation bar: 

<nav>
  <div class="nav-wrapper">
    <a href="#"><div class="logo"></div></a>
    <menu>
      <a href="#"><span>HOME</span></a>
      <a href="#"><span>PROJECTS</span></a>
      <a href="#"><span>ABOUT</span></a>
    </menu>
  </div>
</nav>

The CSS below showcases how I styled the HTML: 

  .nav {
    height: 40px;
    width: 100%;
    position: fixed;
    top: 0;
    left: 0;
    display: flex;
    justify-content: center;
    align-items: center;
    z-index: 2;
  }
  .nav-wrapper {
    max-width: 1280px;
    width: 100%;
    background-color: #303030;
    opacity: 0.97;
    display: flex;
    justify-content: space-between;
  }

Although there there were too many topics that I studied than I can cover in this article, here are some major concepts that I examined which include, but were not limited to, the following:

  1. Semantic HTML
  2. The value in using CSS grid and flexbox over frameworks, such as Bootstrap
  3. The hierarchy of the DOM tree
  4. The order in which a browser parses HTML/CSS and renders content in the browser window
  5. When to use CSS vs. JavaScript when building interactive features (simple vs. complex)
  6. Improved perceived performance with skeleton screens (designing latency stages)
  7. Graceful degradation vs. progressive enhancement

As I underwent this learning process, I grew in my appreciation for and ability to empathize with a developer’s unique perspective, priorities, and challenges in responsive web development. For example, just the other day, I read the following in an article:

In its infancy, [Instagram] was too computationally expensive to allow for both landscape and portrait mode; instead of limiting the interface to one or the other, Instagram’s designers decided to make every photo a square. By being square, you didn’t have to choose anymore. It was a great design decision.

~John Maeda, Wired Magazine

Upon reading it, I realized that development-related concerns had become easier for me to understand and anticipate, which translates into more productive and expedient collaboration with developers because I would not need them to explain as many things to me.

CONCLUSION

The more I coded, the more comfortable I became with code. Strengthen my ability to find answers quickly and independently. Gone were the days when just looking at an HTML document would intimidate me.

Step away from the problem when stuck and frustrated. Use resources at my disposal, such as my community, exercise, eating well, sleeping, create a music playlist (classical music, perfect opportunity to test out a new set of headphones). Building a local and remote community. Helps to get unstuck when you have people around you to bounce off ideas on them. Very helpful whenever I got stuck and frustrated. Stay cool and focused under pressure. It helps to get a different perspective by working on a different project or task which allows the subconciuos mind to tackle the original problem in a creative way in the background.

Research shows numerous benefits in bilingualism, especially in how it improves a person's ability to adapt to change, feel empathy towards others, and creatively solve problems. having the imagination and vision to see alternate approaches and results. I felt something similar switching between designing and coding, like a different part of my brain was activated and a different kind of creativity emerged.
